# 第 19 轮改进文档

## 1. 本轮修改

只修改 MVP 文档中的两个表达：

```text
代码式项目名特判表达
  -> 按项目名分支的特殊逻辑

固定默认工具组表达
  -> seed loop 的初始可选工具
```

## 2. 为什么要改

这两个表达虽然不是设计缺口，但会把读者带回错误方向：

```text
按项目名写特判
固定暴露工具组
```

feng 的核心应该是：

```text
按 self 文件和目标成长，不按项目名特判
按本轮需要选择工具，不默认全量暴露
```

## 3. 不继续扩写

本轮没有发现新的架构概念缺口。

继续扩写会进入实现规格层。
